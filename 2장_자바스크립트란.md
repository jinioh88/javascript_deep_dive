# 내용 정리
## 자바스크립트의 성장의 역사
### Node.js
- 비동기 I/O를 지원하며 단일 스레드 이벤트 루프 기반으로 동작함으로써 요청처리 성능이 좋다. 
- 따라서 Node.js는 데이터를 실시간으로 처리하기 위해 I/O가 빈번하게 발생하는 SPA에 적합하다.
- 하지만 CPU 사용률이 높은 애플리케이션에는 권장하지 않는다. 

## 자바스크립트와 ECMAScript
- 크ㄹ라이언트 사이드 WEB API, 즉 DOM, BOM, Canvas, XxMLHttpRequest, fetch, requestAnimationFrame, Web Component 드ㅇ을 아우르는 개념이다.
- 클라이언트 사이드 WEB API느ㄴ ECMAScript오ㅏ 별도로 WWW 코ㄴ소시엄에서 별도의 사양으로 관리하고 있다.

💬 3️⃣ 프론트엔드 면접 포인트
질문 1: “ECMAScript와 Web API의 차이는 무엇인가요?”
답변 포인트:
ECMAScript: 언어의 핵심 문법과 기능 (변수, 함수, 클래스, 비동기 등)
Web API: 브라우저가 제공하는 환경별 기능 (DOM, Fetch, localStorage 등)
비유:
ECMAScript = 언어의 문법
Web API = 운영체제의 기능
질문 2: “Node.js와 브라우저 환경에서의 JS 차이점은?”
핵심 차이:
구분	브라우저	Node.js
실행 주체	브라우저 엔진 (V8, SpiderMonkey 등)	V8 + libuv
전역 객체	window, document	global, process
주요 역할	UI 조작, 사용자 인터랙션	서버, 파일, 네트워크 처리
I/O 모델	제한적 (보안 이유)	비동기 I/O 가능
면접 포인트:
“Node.js에서 DOM 조작이 안 되는 이유는?”
→ Node.js는 브라우저 환경이 아니므로 DOM API를 제공하지 않음.

🎯 미니 프로젝트 아이디어
🌐 “간단한 서버 vs 브라우저 환경 비교”
Node.js로 간단한 서버를 만들어보고, 브라우저 콘솔에서도 동일한 로직을 테스트해보세요.
``` js
// Node.js: simple server
const http = require("http");

http.createServer((req, res) => {
  res.write("Hello from Node.js!");
  res.end();
}).listen(3000);
```
``` js
// Browser: Fetch the response
fetch("http://localhost:3000")
  .then(res => res.text())
  .then(console.log);
```
동일한 언어(JS)로 다른 런타임을 경험해보는 실습은 “환경 의존성”을 이해하는 데 매우 도움이 됩니다.
