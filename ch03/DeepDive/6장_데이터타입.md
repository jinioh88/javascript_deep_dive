# 내용정리
## 숫자 타입
- ECMAScript 사양에 따르면 숫자 타입의 값은 배정밀도 64비트 부동소수점 형식을 따른다. 
- 자바스크립트는 2진수, 8진수, 16진수를 표현하기 위한 데이터 타입을 제공하지 않기 때문에 이들 값을 참조하면 모두 10진수로 해석된다. 

## 문자열 타입
- 자바스크립트의 문자열은 원시 타입이며, 변경 불릉한 값이다. 

## 템플릿 리터럴
### 멀티라인 문자열
- 이스케이프 시퀀스를 사용하지 않고도 줄바꿈 허용. 공백도 있는 그대로 적용.

### 표현식 삽입
- ${}으로 표현해 표현식 삽입.
- 표현식의 평가 결과가 문자열이 아니더라도 문자열로 타입이 강제 변환되어 삽입
- 표현식 삽입은 반드시 템플릿 리터럴 내에서 사용해야 함.

## 심벌 타입
- 다른 값과 중복되지 않는 유일무이한 값.
- 주로 이름이 충돌할 위험이 없는 객체의 유일한 프로퍼티 키를 만들기 위해 사용.
- Symbol 홈수를 호출해 생성. 생성된 심벌값은 외부에 노출되지 않으며, 다른 값과 절대 중복되지 않는 유일무이한 값.

## 데이터 타입의 필요성
### 데이터 타입에 의한 메모리 공간의 확보와 참조
- 자바스크립트 엔진은 데이터 타입, 즉 값의 종류에 따라 정해진 크기의 메모리 공간을 확보한다. 

## 동적 타이핑
### 동적 타입 언어와 정적 타입 언어
- 정적 타입 언어는 컴파일 시점에 타입 체크를 수행. 
- 자바스크립트의 변수는 선언이 아닌 할당에 의해 타입이 결정된다. 
- 재할당에 의해 변수의 타입은 언제든 동적으로 변할 수 있다. 이를 동적 타입 언어라 함.

### 동적타입 언어와 변수
- 변수는 꼭 필요한 경우에 한해 제한적으로 사용.
- 변수의 유효 범위는 최대한 좁게 만들어 변수의 부작용을 억제해야 한다. 
- 전역 변수는 최대한 사용하지 않도록 한다. 
- 변수보다는 상수를 사용해 값의 변경을 억제한다. 
- 변수 이름은 변수의 목적이나 의미를 파악할 수 있도록 네이밍한다.

💻 2️⃣ 코드 실습 문제
🧩 문제 1. 숫자 타입의 특징 실험
``` js
console.log(0.1 + 0.2 === 0.3); // ?

// 결과가 false인 이유를 설명해보세요.

```
💡 JS의 모든 숫자는 IEEE 754 64비트 부동소수점 형식으로 저장됩니다.
이 때문에 정확히 표현할 수 없는 2진 부동소수점 오차가 발생합니다.

🧩 문제 2. 문자열 불변성 실험
``` js
let str = "Hello";
str[0] = "J";

console.log(str); // ?
```
💡 문자열은 불변값이므로, 내부 문자를 직접 변경할 수 없습니다.
이 코드의 결과는 여전히 "Hello"입니다.
새로운 문자열을 만들 때마다 새 메모리 공간이 할당됩니다.

🧩 문제 3. 템플릿 리터럴 실습
``` js
const name = "세진";
const score = 95;

console.log(`학생 이름: ${name}
점수: ${score}
합격 여부: ${score >= 60 ? "합격" : "불합격"}`);

```
✅ 템플릿 리터럴은 문자열 연결(+)보다 가독성이 좋고, 줄바꿈과 표현식 평가를 지원합니다.

🧩 문제 4. Symbol 고유성 확인
``` js
const id1 = Symbol("userId");
const id2 = Symbol("userId");

console.log(id1 === id2); // ?

```
결과는 false.
Symbol() 로 생성된 값은 설명(description)이 같더라도 항상 고유(unique) 합니다.

🧩 문제 5. 동적 타이핑 확인
``` js
let value = 10;      // number
value = "Genie";     // string
value = { score: 80 } // object

console.log(typeof value);

```
💡 자바스크립트 변수는 선언이 아닌 할당 시점에 타입이 결정되며,
재할당으로 언제든 타입이 바뀔 수 있음.

💬 3️⃣ 프론트엔드 면접 포인트

질문 1: “자바스크립트의 숫자 타입은 왜 하나뿐인가요?”

핵심 포인트:

JS의 숫자 타입은 모두 배정밀도 64비트 부동소수점(double precision floating point) 형식.

정수, 실수를 구분하지 않음 → 계산 편의성은 높지만 정밀도 오류 발생 가능.

추가 질문:

“0.1 + 0.2가 0.3이 아닌 이유를 설명하세요.”

→ 0.1, 0.2는 2진수로 정확히 표현 불가 → 부동소수점 반올림 오차 발생.


질문 2: “문자열이 불변값이라는 건 무슨 뜻인가요?”

답변 포인트:

한 번 생성된 문자열은 변경 불가, 수정하려면 새로운 문자열이 생성됨.

React에서 상태(state) 업데이트 시 “불변성 유지”의 근거로 자주 언급됨.


질문 3: “Symbol 타입은 언제 사용하나요?”

핵심 포인트:

객체 프로퍼티 키 충돌 방지용.

예: 라이브러리나 프레임워크 내부에서 “은닉 키(hidden key)”를 만들 때 사용.

예시:
``` js
const TOKEN = Symbol("token");
const user = { [TOKEN]: "secret-value" };
console.log(Object.keys(user)); // []
console.log(user[TOKEN]); // secret-value

```
