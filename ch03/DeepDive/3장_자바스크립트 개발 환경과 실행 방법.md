# 내용정리
## 자바스크립트 실행 환경
- 모든 브라우저는 자바스크립트를 해석하고 실행할 수 있는 자바스크립트 엔진을 내장하고 있음. 
- Node.js도 자바스크립트 엔진을 내장. 
- 브라우저는 HTML, CSS, 자바스크립트를 실행해 웹페이지를 브라우저 화면에 랜더링하는 것이 주된 목적.
- Node.js는 브라우저 외부에서 자바스크립트 실행화경을 제공하는 것이 주된 목적.

💻 2️⃣ 코드 실습 문제
이번 장은 실행 환경의 차이를 직접 관찰해보는 실습이 좋습니다.

🧩 문제 1. 브라우저 vs Node.js 전역 객체 차이
아래 코드를 각각 브라우저 콘솔과 Node.js 콘솔에서 실행해보세요.
``` js
console.log(this);
console.log(globalThis);
```
질문:

브라우저에서는 무엇이 출력되나요?

Node.js에서는 무엇이 출력되나요?

💡 포인트:
브라우저 → window
Node.js → global
둘 다 ES2020 이후에는 globalThis로 통일됨.

💬 3️⃣ 프론트엔드 면접 포인트

질문 1: “브라우저와 Node.js의 자바스크립트 실행 환경 차이는 무엇인가요?”

핵심 비교

구분	브라우저	Node.js
목적	UI 렌더링, 사용자 인터랙션	서버, CLI, 백엔드 로직
전역 객체	window, document	global, process
실행 엔진	V8, SpiderMonkey 등	V8 + libuv
API 제공자	Web API (DOM, Fetch 등)	Node API (fs, net 등)
이벤트 루프	브라우저 루프 (Web APIs, Task Queue)	Node 이벤트 루프 (libuv)

질문 2: “렌더링 엔진과 자바스크립트 엔진의 관계는 무엇인가요?”

답변 포인트:

렌더링 엔진(예: Blink, WebKit)은 HTML/CSS를 처리.

JS 엔진(예: V8)은 JS 코드를 해석하고 실행.

브라우저 내부에서 두 엔진이 협력하여 화면을 완성함.

예시:
JS가 DOM을 조작하면 → 렌더링 엔진이 다시 그린다 (repaint/reflow).

질문 3: “Node.js에는 DOM이 없는데, React 서버사이드 렌더링은 어떻게 가능한가요?”

핵심 설명:

Node.js는 브라우저가 아니므로 DOM 조작은 불가.

대신 ReactDOMServer 같은 라이브러리가 HTML 문자열을 생성.

실제 브라우저 렌더링은 클라이언트 쪽에서 수행됨.
