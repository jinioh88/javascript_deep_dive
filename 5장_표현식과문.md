# 내용정리
## 값
- 값은 식(표현식)이 평가되어 생성된 결과를 말한다. 

## 리터럴
- 리터럴은 사람이 이해할 수 있는 문자 또는 약속된 기호를 사용해 값을 생성하는 표기법을 말한다.
- 자바스크립트 엔진은 코드가 런타임에 리터럴을 평가해 값을 생성한다. 

## 표현식
- 표현식은 값으로 평가될 수 있는 문이다. 
- 표현식이 평가되면 새로운 값을 생성하거나 기존 값을 참조한다. 

## 문
- 문은 프로그램을 구성하는 기본 단위이자 최소 실행 단위다. 
- 문은 명령문이라고도 부른다. 

## 표현식인 문과 표현식이 아닌 문
- 표현식인 문과 표현식이 아닌 문을 구별하는 가장 간단하고 명료한 방법은 변수에 할당해 보는 것이다. 
- 표현식인 문은 값으로 평가되므로 변수에 할당할 수 있다. 


# 추가정리
🧠 1️⃣ 핵심 복습 문제

값(value) 이란 무엇이며, 자바스크립트에서 어떻게 만들어지나요?

리터럴(literal) 과 표현식(expression) 의 차이를 설명해보세요.

문(statement) 은 무엇이며, 표현식과 어떻게 다른가요?

표현식인 문과 표현식이 아닌 문을 구분하는 방법은?

평가(evaluation) 라는 개념은 자바스크립트에서 어떤 의미를 갖나요?

💻 2️⃣ 코드 실습 문제
🧩 문제 1. 표현식 vs 문 구분하기

아래 코드 중 표현식과 표현식이 아닌 문을 구분해보세요 👇
``` js
// A
let x;

// B
x = 10;

// C
if (x > 5) {
  console.log("x는 5보다 큼");
}

// D
x + 5;
```
해설 포인트

let x; → 표현식이 아닌 문 (값이 평가되지 않음)

x = 10; → 표현식인 문 (값 10이 평가되어 x에 할당됨)

if (x > 5) {...} → 표현식이 아닌 문

x + 5; → 표현식 (평가 결과는 있지만 명령이 아님)


🧩 문제 2. 표현식 여부 판별 실험

표현식인 문과 표현식이 아닌 문을 구분하는 가장 확실한 방법은 “변수에 할당해보기”입니다.
``` js
let test = (x = 5);
console.log(test); // ?

let test2 = (let y = 10); // ?
```
질문:
1️⃣ 첫 번째 줄은 정상 동작하지만
2️⃣ 두 번째 줄은 문법 에러가 납니다. 왜 그럴까요?

💡 이유: let y = 10;은 선언문(statement) 으로, 값으로 평가되지 않기 때문입니다.

🧩 문제 3. 표현식의 평가 결과 실험
``` js
const a = 5 * 2;         // ?
const b = a > 8;         // ?
const c = b ? "Big" : "Small"; // ?
console.log(a, b, c);
```
위 코드는 표현식의 연속적인 평가 흐름을 보여줍니다.
각 표현식이 평가될 때 어떤 값이 만들어지는지 콘솔에 출력해보세요.

💬 3️⃣ 프론트엔드 면접 포인트

질문 1: “표현식과 문은 자바스크립트에서 어떻게 구분되나요?”

핵심 포인트:

표현식(expression): 값으로 평가되는 코드

문(statement): 명령을 수행하는 코드

예시:
``` js
let a;        // 문 (선언)
a = 10;       // 표현식인 문
a + 5;        // 표현식
if (a > 10) { // 문
  console.log(a);
}
```

질문 2: “표현식의 평가 결과는 어디에 사용되나요?”

답변 요약:

표현식의 결과는 새로운 값으로 사용되거나, 다른 표현식의 일부로 결합됩니다.

React의 JSX 문법도 내부적으로 표현식 기반이에요.

예시 (React)
``` js
const name = "Genie";
return <h1>Hello {name}</h1>; // 중괄호 안에는 항상 '표현식'만 올 수 있다!
```

질문 3: “세미콜론(;)의 의미는 무엇인가요?”

핵심 설명:

세미콜론은 문(statement)의 종료를 나타냄.

자바스크립트는 “ASI(Automatic Semicolon Insertion)” 규칙으로 세미콜론을 자동 삽입하기도 함.

하지만 예상치 못한 동작이 생길 수 있어, 팀 컨벤션에 따라 명시하는 것을 권장.
